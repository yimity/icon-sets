# icon-sets
# 解析图标集
    1. 解析 collection.json 文件，找到所有的图标集合。
    2. 查找两次 git 提交之间变更的 json 文件夹中 .json 结尾的文件，
    3. 过滤 .json 文件列表，collection.json 中图标集 hidden 不为 true 的。 即两次结果的交集，得到需要更新的图标的集合。
    4. 解析各个 .json 文件，获得图标集的信息，包含图标的具体数据。最新的。
    5. 解析各个 .json 文件，获得图标集的信息，包含图标的具体数据。上一次提交的。
    6. 通过判断文件存在与否，两次都是 true 说明是修改；当前是 true，上一次是 false, 说明是增加，当前是 false，上一次是 true, 说明是删除。

# 解析图标数据
    1. 遍历图标库。
    2. 如果图标库是删除状态，则其所有的图标都是删除。
    3. 如果图标库是增加状态，则其所有的图标都是增加状态。
    4. 如果图标库是修改状态，通过两次 git 记录中图标的变更(逻辑和图标集一致)，
        4.1 找出所有添加的新图标，添加。
        4.2 找出所有删除的图标，删除。
        4.3 通过 git 两次提交记录中，图标的内容对比，找出修改的图标。

# 写库
    1. 新增的图标集 
        1.1. 创建 Project 对应图标集。
        1.2. 创建图标内容。
        1.3. 创建 Project 和 图标的关系。
    2. 删除的图标集，暂时不处理，后续逻辑为：
        2.1 如果图标没有被使用，删除。
        2.2 如果图标有被使用，则不处理。  
        2.3 如果图标都被删除了，则删除图标集。
    3. 修改的图标集
        3.1 新增的图标，新增。
        3.2 删除的图标，逻辑同删除的图标库。
        3.3 修改的图标。 
            3.3.1 如果图标没有被使用，修改。
            3.3.2 如果图标有被使用，则不处理。同时新增一个同名+_+lastModified的图标。
